# Production Domain Migration — mymathroots.com Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `https://mymathroots.com` the official production domain everywhere it is referenced in the codebase, replacing all stale `mymathroots.netlify.app` and `mymathroots.app` references in public-facing and config files.

**Architecture:** This is a text-replacement and metadata task only — no behaviour changes. Source files (`src/`, `netlify/functions/`, `supabase/`) are updated where they reference the public domain in user-facing copy, VAPID config, and documentation. SEO canonical + Open Graph tags are added to `index.html`. A `robots.txt` is created. The build is run to regenerate `dist/`, and the test suite verifies no regressions.

**Tech Stack:** Vanilla JS SPA, Netlify Functions (Node 18), Supabase Edge Functions (Deno), Jest test suite, `node build.js` build pipeline.

### Netlify URLs intentionally kept

`mymathroots.netlify.app` and `mymathroots-staging.netlify.app` are retained in the CORS `ALLOWED_ORIGINS` arrays in both Netlify functions. Netlify continues to serve those hostnames even after a custom domain is set (unless force-redirected), so removing them from CORS now would break any cached service worker or link still pointing there. They can be removed after a full traffic migration soak period.

### Build output — not edited directly

`dist/` is in `.gitignore` and is regenerated by `node build.js`. After `privacy.html` is fixed, the build regenerates `dist/privacy.html` automatically (Task 6 verifies this).
