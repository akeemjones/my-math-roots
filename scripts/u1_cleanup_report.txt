3. New Heuristic Categories Added
----------------------------------
classifyWordDistractor() function added to scripts/migrate_u1_phase2.js,
called when wn/cn parse fails (i.e., word/expression distractors). Twelve
heuristic blocks, first-match-wins:

  1.   Strategy-pick prompt + strategy-phrase distractor → err_wrong_operation
  2.   Strategy-pick prompt + arithmetic-expression distractor → err_wrong_operation
  2.5  Strategy-pick prompt + both options strategy phrases → err_wrong_operation
  3.   Doubles helper / "X+X" prompt → err_doubles_confusion
  4.   Make-ten helper prompt / "+10/-10" pattern → err_make_ten_confusion
  4.5  Pair-format "X and Y" with one number off by 1 → err_off_by_one
  5.   Fact-family arithmetic with reversed/related operation → err_fact_family_confusion
  6.   Compare-number-sentence prompt → err_fact_family_confusion
  7.   Reversed-digit answer → err_reversed_digits
  8.   Number-line direction confusion → err_inverse_confusion
  9.   Mistake-finder / yes-no-explanation reasoning → err_concept_confusion
  10.  u1l4 strategy-phrase distractor → err_fact_family_confusion
  11.  u1l1 generic count-strategy fallback → err_wrong_operation
  12.  u1l3 strategy-phrase or arithmetic-expression distractor → err_make_ten_confusion

The migration is now idempotent: extractVal() handles already-object-form
options, allowing safe re-runs without index drift.

Sample (u1l1 — "pick the matching number sentence" pattern):
  Q: Which number sentence can you solve by counting back 2?
     C: 7 - 2     W: 5 + 2 / 3 + 3 / 2 + 2
  Q: Which problem can be solved by counting up from 8?
     C: 8 + 2     W: 8 - 3 / 3 - 8

These remaining items are number-sentence-pick prompts where the wrong
options are arithmetic expressions (not strategy phrases or verbs), so they
fall outside heuristics 1–12. They are genuinely hard concept distractors —
each wrong option is a valid math expression that just doesn't match the
strategy described in the prompt. Tagging them more specifically would
require a new heuristic that detects "expression in option matches/doesn't
match operation in prompt", which is brittle. Documented and shipped at
6.05% rather than over-engineered.

Sample (u1l2/u1l3 — yes-no-explanation reasoning):
  Q: Jake says 6 + 7 = 14 because double 7 is 14. Is Jake correct?
     C: No, it should be 13     W: No, it should be 12
  Q: A student wrote: 7 + 6 = 14. ... What did they do wrong?
     C: They broke apart 6 into 3 and 4 instead of 3 and 3
     W: They should have started with 6

These are explanation-pick options where every choice begins with the same
phrase ("No, it should be...", "They..."), so the strategy/concept regex
can't distinguish them. Genuine concept distractors.
